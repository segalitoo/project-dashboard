# Project Dashboard

## Project Overview
A dark-themed status dashboard that tracks all local development projects in `~/Claude`. Deployed to GitHub Pages for mobile access.

## Tech Stack
- **Frontend:** Static HTML + CSS + vanilla JS (no framework)
- **Data:** `projects-status.json` generated by scanner script
- **Scanner:** Node.js (`scan-projects.mjs`) — reads git, Claude Code sessions
- **Hosting:** GitHub Pages
- **Automation:** macOS LaunchAgent (every 3 hours) + Claude Code skill

## File Structure
```
project-dashboard/

├── app.js                  # Client-side rendering + filtering

├── CLAUDE.md               # This file
└── .github/
    └── workflows/

```

## Commands
```bash
npm run scan         # Scan projects → generate JSON
npm run scan:push    # Scan + commit + push to GitHub
```

## Key Conventions
- The scanner reads from `~/.claude/projects/` for Claude Code session data
- `projects-status.json` is checked into git and served as static data
- The dashboard uses flat HTML/CSS/JS — no build step needed
- LaunchAgent at `~/Library/LaunchAgents/com.rannsegal.project-dashboard.plist` runs scan:push every 3 hours

## How Data Flows
1. `scan-projects.mjs` scans all sibling directories in `~/Claude`
2. For each project: reads git data, Claude Code sessions, metadata
3. Outputs `projects-status.json` with status (active/idle/stale)
4. `npm run scan:push` commits and pushes to GitHub
5. GitHub Pages serves the static site with the latest JSON
